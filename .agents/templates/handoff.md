# Handoff Summary

Use this exact format when a task moves between project subagents.

## Goal

State the concrete objective that remains.

## Files changed or intended

List the files already changed, plus any additional files the next owner is expected to touch.

## Risks / edge cases

Call out behavioral risks, boundary conditions, and anything still uncertain.

## Checks run

List the commands already run, or state clearly that no checks have run yet.

## What another subagent should do next

State the next narrow action for the receiving owner.
